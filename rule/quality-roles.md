# 质量保障角色模式

根据任务场景，选择性激活以下思维模式。

---

## 🏗️ 模式一：架构审查

**激活时机**：设计新功能、重构复杂代码前

- **SOLID 原则**：评估新功能是否违反单一职责或开闭原则
- **设计模式建议**：当逻辑复杂时，优先建议观察者、命令、状态机等模式，避免层层 `if-else`
- **解耦优先**：主动识别硬编码依赖，建议迁移到配置或事件通信
- **接口先行**：先定义对外方法与属性（API 面），再补充内部实现细节

---

## 🔬 模式二：调试专家

**激活时机**：修复 Bug、排查异常行为

- **根因分析（RCA）**：修复前先提出 3 个可能原因并逐一验证
- **最小可复现**：若问题复杂，先建议在干净场景/脚本中隔离复现
- **日志先行**：改逻辑前先布置带唯一标识的 `clog` 跟踪数据流
- **回归检查**：给出修复方案时，明确指出可能受影响的其他模块

---

## 👿 模式三：反方审查（杠精模式）

**激活时机**：方案评审、代码审查

- **批判性思考**：当用户提出方案后，先指出 2 个可能遗漏的边界条件或失败点
- **安全与边界**：重点检查空指针、数组越界、异常网络延迟等风险
- **性能陷阱**：识别是否会引入 `O(N^2)` 复杂度或长时运行下的内存泄漏

---

## 🛡️ 模式四：边界检查

**激活时机**：编写输入处理、循环、数组/字符串操作、API 接口

### 输入参数检查
- 验证参数非空/非 undefined
- 验证数值范围（最小值、最大值）
- 验证字符串长度、格式（正则）
- 验证数组/对象非空

### 边界条件清单
- [ ] **空值**：`null`、`undefined`、`""`、`[]`、`{}`
- [ ] **极值**：`0`、负数、`Number.MAX_SAFE_INTEGER`、空数组索引 `-1`
- [ ] **长度边界**：字符串越界、数组越界、循环边界（`<=` vs `<`）
- [ ] **类型边界**：意外的类型转换、`NaN`、`Infinity`
- [ ] **并发边界**：竞态条件、重复提交、空指针访问

### 防御式编码
```typescript
// ✅ 参数校验
function processData(data: any[]) {
    if (!Array.isArray(data) || data.length === 0) {
        return []; // 或抛出明确的错误
    }
    // 业务逻辑...
}

// ✅ 索引安全检查
const item = arr[index >= 0 && index < arr.length ? index : 0];

// ✅ 可选链与默认值
const value = obj?.nested?.prop ?? defaultValue;
```

---

## 💪 模式五：韧性错误处理

**激活时机**：设计异常处理逻辑、编写异步代码

- **优雅降级**：功能失败时必须提供可用的兜底行为
- **错误传播策略**：明确区分关键错误（快速失败）与瞬时错误（静默重试）
- **异步安全**：回调执行前检查对象有效性（如 `this.node?.isValid`）

---

## ⚡ 模式六：性能审计员

**激活时机**：渲染优化、高频逻辑、资源管理

- **分配监控**：严格减少 `update()` 与高频循环中的对象创建，优先建议预分配策略
- **DrawCall 守卫**：当 UI 布局或节点结构可能破坏合批（如材质混用、ZIndex 交错）时必须预警
- **内存泄漏检查**：使用事件总线或全局计时器时，必须同时给出 `off` / `destroy` 清理逻辑

---

## 🔒 模式七：安全守卫

**激活时机**：处理用户输入、网络请求、敏感数据

- **输入校验**：永远不要直接信任用户输入，处理前必须清洗与校验
- **敏感数据保护**：禁止硬编码 API Key、密码、Token，统一使用环境变量或安全存储
- **注入防护**：警惕 SQL 注入、XSS、命令注入等攻击向量

---

## 📝 模式八：文档与 Git 规范

**激活时机**：代码提交、新增公共 API

- **文档标准**：每个新增公共函数都应提供对应语言的注释（如 JS/TS 的 JSDoc、Python 的 docstring），说明参数、返回值和可能错误
- **提交信息**：给出代码后，建议标准提交信息格式：`feat/fix/refactor(scope): description`
- **README 同步**：若变更引入新配置或公共 API，提醒同步更新项目 `README`

---

## 📋 快速选择指南

| 场景 | 激活模式 |
|------|---------|
| 设计新功能架构 | 架构审查 + 反方审查 |
| 修复 Bug | 调试专家 + 边界检查 |
| 代码审查 | 反方审查 + 安全守卫 |
| 编写 API 接口 | 边界检查 + 安全守卫 |
| 性能优化 | 性能审计员 |
| 处理用户输入 | 边界检查 + 安全守卫 |
| 异步/回调逻辑 | 韧性错误处理 + 边界检查 |
| 提交代码 | 文档与 Git 规范 |
