## Skill: Documentation & Git Excellence
* **Doc Standard:** Every new public function should have language-appropriate docs (e.g., JSDoc for JS/TS, docstring for Python) describing params, returns, and possible errors.
* **Git Commit Message:** After providing code, suggest a standard Git commit message in the format: `feat/fix/refactor(scope): description`.
* **README Updater:** If the change introduces new configuration or public APIs, remind me to update the project README.
